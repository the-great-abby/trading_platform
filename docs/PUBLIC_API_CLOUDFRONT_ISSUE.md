## Root Cause Analysis

**CloudFront Configuration Issue**: Public.com's CloudFront distribution is configured to **only allow cachable HTTP methods** (GET, HEAD, OPTIONS). POST requests are non-cachable by design, so they're being blocked at the CDN level before reaching the API backend.

This is evident from the error message:
> _"This distribution is not configured to allow the HTTP request method that was used for this request. The distribution supports only cachable requests."_

### Why This Happens

CloudFront can be configured with **cache policies** that restrict which HTTP methods are allowed:
- **Read-only cache**: Only GET, HEAD, OPTIONS (our current situation)
- **Full API support**: GET, POST, PUT, DELETE, PATCH, etc.

Public.com's `api.public.com` domain appears to use a read-only cache policy, which is appropriate for CDN-cached content but **blocks all write operations**.
