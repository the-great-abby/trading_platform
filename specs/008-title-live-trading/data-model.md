# Data Model: Live Trading System

**Feature**: 008-title-live-trading  
**Date**: 2025-09-25  
**Status**: Complete

## Entity Overview

Based on the feature specification, this data model defines the entities required for live trading with Public.com API integration.

## Core Entities

### 1. Live Trading Account
**Purpose**: Represents a Public.com brokerage account with real money

**Fields**:
- `account_id` (UUID, Primary Key)
- `public_account_id` (String, Unique) - Public.com account identifier
- `account_name` (String) - Human-readable account name
- `account_type` (Enum: CASH, MARGIN) - Account type
- `buying_power` (Decimal) - Available buying power
- `cash_balance` (Decimal) - Cash balance
- `equity` (Decimal) - Total account equity
- `is_active` (Boolean) - Account active status
- `created_at` (DateTime)
- `updated_at` (DateTime)

**Validation Rules**:
- Account ID must be unique across all live trading accounts
- Public account ID must be valid Public.com account format
- Buying power and balances must be non-negative

### 2. Live Position
**Purpose**: Represents an actual position held in the brokerage account

**Fields**:
- `position_id` (UUID, Primary Key)
- `account_id` (UUID, Foreign Key) - Links to Live Trading Account
- `symbol` (String) - Trading symbol (e.g., AAPL, AAPL_250C_20241220)
- `strategy` (Enum: IRON_CONDOR, BUTTERFLY_SPREAD, CALENDAR_SPREAD)
- `quantity` (Integer) - Position size (positive=long, negative=short)
- `average_price` (Decimal) - Average entry price
- `current_price` (Decimal) - Current market price
- `unrealized_pnl` (Decimal) - Unrealized profit/loss
- `max_risk` (Decimal) - Maximum risk for this position
- `status` (Enum: ACTIVE, CLOSED, EXPIRED)
- `opened_at` (DateTime)
- `closed_at` (DateTime, Nullable)
- `updated_at` (DateTime)

**Validation Rules**:
- Quantity must be non-zero
- Average price must be positive
- Status transitions: ACTIVE → CLOSED/EXPIRED only

### 3. Live Trade
**Purpose**: Represents an executed trade through Public.com API

**Fields**:
- `trade_id` (UUID, Primary Key)
- `account_id` (UUID, Foreign Key) - Links to Live Trading Account
- `position_id` (UUID, Foreign Key, Nullable) - Links to Live Position
- `public_order_id` (String, Unique) - Public.com order identifier
- `symbol` (String) - Trading symbol
- `strategy` (Enum: IRON_CONDOR, BUTTERFLY_SPREAD, CALENDAR_SPREAD)
- `action` (Enum: BUY, SELL) - Trade action
- `quantity` (Integer) - Trade quantity
- `price` (Decimal) - Execution price
- `total_value` (Decimal) - Total trade value
- `commission` (Decimal) - Commission paid
- `status` (Enum: PENDING, FILLED, PARTIALLY_FILLED, REJECTED, CANCELLED)
- `fill_quantity` (Integer) - Actual filled quantity
- `average_fill_price` (Decimal) - Average fill price
- `executed_at` (DateTime, Nullable)
- `created_at` (DateTime)
- `updated_at` (DateTime)

**Validation Rules**:
- Order ID must be unique across all trades
- Quantity must be positive
- Price must be positive
- Status transitions follow order lifecycle

### 4. Risk Profile
**Purpose**: Represents risk parameters and limits for live trading

**Fields**:
- `profile_id` (UUID, Primary Key)
- `account_id` (UUID, Foreign Key) - Links to Live Trading Account
- `max_position_size` (Decimal) - Maximum position size as % of portfolio
- `max_portfolio_risk` (Decimal) - Maximum portfolio risk as % of capital
- `max_daily_loss` (Decimal) - Maximum daily loss as % of capital
- `max_daily_trades` (Integer) - Maximum trades per day
- `position_size_limit` (Decimal) - Absolute position size limit
- `emergency_stop_enabled` (Boolean) - Emergency stop functionality
- `created_at` (DateTime)
- `updated_at` (DateTime)

**Validation Rules**:
- All percentage values must be between 0 and 1
- Daily trade limit must be positive
- Emergency stop can only be disabled by admin

### 5. API Credentials
**Purpose**: Secure storage of Public.com authentication tokens

**Fields**:
- `credential_id` (UUID, Primary Key)
- `account_id` (UUID, Foreign Key) - Links to Live Trading Account
- `access_token` (String, Encrypted) - Public.com access token
- `token_type` (String) - Token type (Bearer)
- `expires_at` (DateTime) - Token expiration
- `refresh_token` (String, Encrypted, Nullable) - Refresh token if available
- `is_active` (Boolean) - Credential active status
- `created_at` (DateTime)
- `updated_at` (DateTime)

**Validation Rules**:
- Access token must be valid JWT format
- Expiration must be in the future
- Only one active credential per account

### 6. Trade Signal
**Purpose**: Represents trading decisions generated by strategies

**Fields**:
- `signal_id` (UUID, Primary Key)
- `account_id` (UUID, Foreign Key) - Links to Live Trading Account
- `strategy` (Enum: IRON_CONDOR, BUTTERFLY_SPREAD, CALENDAR_SPREAD)
- `symbol` (String) - Target symbol
- `action` (Enum: BUY, SELL, HOLD) - Recommended action
- `quantity` (Integer) - Recommended quantity
- `target_price` (Decimal, Nullable) - Target execution price
- `confidence` (Decimal) - Signal confidence (0-1)
- `market_conditions` (JSON) - Market condition data
- `status` (Enum: PENDING, EXECUTED, REJECTED, EXPIRED)
- `executed_trade_id` (UUID, Foreign Key, Nullable) - Links to executed trade
- `generated_at` (DateTime)
- `expires_at` (DateTime)
- `created_at` (DateTime)

**Validation Rules**:
- Confidence must be between 0 and 1
- Target price must be positive if provided
- Expiration must be after generation time

### 7. Order Status
**Purpose**: Tracks the state of orders submitted to Public.com

**Fields**:
- `status_id` (UUID, Primary Key)
- `trade_id` (UUID, Foreign Key) - Links to Live Trade
- `public_order_id` (String) - Public.com order identifier
- `status` (Enum: PENDING, SUBMITTED, FILLED, PARTIALLY_FILLED, REJECTED, CANCELLED)
- `status_message` (String, Nullable) - Status description
- `fill_price` (Decimal, Nullable) - Fill price if filled
- `fill_quantity` (Integer, Nullable) - Fill quantity if filled
- `remaining_quantity` (Integer, Nullable) - Remaining quantity
- `timestamp` (DateTime)
- `created_at` (DateTime)

**Validation Rules**:
- Status transitions follow order lifecycle
- Fill data only present when status is FILLED or PARTIALLY_FILLED
- Timestamp must be accurate to second

## Relationships

### Primary Relationships
- Live Trading Account (1) → (N) Live Positions
- Live Trading Account (1) → (N) Live Trades
- Live Trading Account (1) → (1) Risk Profile
- Live Trading Account (1) → (1) API Credentials
- Live Trading Account (1) → (N) Trade Signals

### Secondary Relationships
- Live Position (1) → (N) Live Trades
- Live Trade (1) → (N) Order Statuses
- Trade Signal (1) → (0..1) Live Trade

## State Transitions

### Order Lifecycle
```
PENDING → SUBMITTED → FILLED
    ↓         ↓         ↓
REJECTED  CANCELLED  PARTIALLY_FILLED → FILLED
```

### Position Lifecycle
```
ACTIVE → CLOSED
   ↓        ↓
EXPIRED  (via time)
```

### Signal Lifecycle
```
PENDING → EXECUTED → (Trade Created)
   ↓         ↓
EXPIRED  REJECTED
```

## Indexes

### Performance Indexes
- `idx_live_positions_account_symbol` on (account_id, symbol)
- `idx_live_trades_account_created` on (account_id, created_at)
- `idx_live_trades_public_order` on (public_order_id)
- `idx_trade_signals_account_strategy` on (account_id, strategy)
- `idx_order_status_trade` on (trade_id)

### Unique Indexes
- `uniq_live_accounts_public_id` on (public_account_id)
- `uniq_live_trades_public_order` on (public_order_id)
- `uniq_api_credentials_account` on (account_id) WHERE is_active = true

## Data Retention

### Retention Policies
- **Live Trades**: 7 years (regulatory requirement)
- **Live Positions**: 7 years (regulatory requirement)
- **Trade Signals**: 1 year (operational requirement)
- **Order Status**: 7 years (regulatory requirement)
- **API Credentials**: 90 days after deactivation

### Cleanup Procedures
- Automated cleanup runs daily
- Soft delete for audit trail preservation
- Secure deletion of sensitive data
- Compliance reporting before deletion

## Security Considerations

### Encryption
- API Credentials: Encrypted at rest using AES-256
- Sensitive fields: Token fields encrypted
- Database: TDE (Transparent Data Encryption) enabled

### Access Control
- Row-level security on account_id
- API access requires valid authentication
- Audit logging for all data access
- Role-based permissions for different user types

## Compliance Features

### Audit Trail
- All entity changes logged with timestamps
- User identification for all modifications
- Before/after values for critical fields
- Immutable audit log storage

### Regulatory Reporting
- Trade execution reports
- Position reports
- Risk management reports
- Compliance violation reports

## Performance Considerations

### Partitioning
- Live Trades: Partitioned by created_at (monthly)
- Order Status: Partitioned by trade_id
- Trade Signals: Partitioned by generated_at (monthly)

### Caching
- Active positions cached in Redis
- Account balances cached with TTL
- Risk profiles cached per account
- Market data cached with expiration

This data model provides a comprehensive foundation for live trading operations while maintaining regulatory compliance and performance requirements.
